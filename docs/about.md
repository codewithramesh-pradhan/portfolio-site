# About Me

I'm a cloud and infrastructure engineer with hands-on experience designing, deploying, and maintaining systems on AWS. My work spans cloud architecture, Linux administration, container orchestration, networking, and automation.

## What I Do

- Design and implement cloud infrastructure on AWS using services like EC2, S3, RDS, CloudFront, and CloudFormation
- Build and manage containerized workloads with Docker and Kubernetes
- Automate provisioning, deployments, and operational tasks using IaC and scripting
- Monitor, troubleshoot, and resolve infrastructure incidents across distributed systems
- Write clear technical documentation for runbooks, architecture decisions, and post-mortems

## How I Work

I approach infrastructure the same way a developer approaches code: version-controlled, tested, peer-reviewed, and documented. I believe in:

- **Automation over manual processes** — If I do it twice, I script it
- **Observability by default** — Logging, metrics, and alerting from day one
- **Incremental improvement** — Ship small changes, measure impact, iterate
- **Clear documentation** — Future me (and the team) will thank present me

## Background

My path into cloud engineering started with Linux system administration and homelab experimentation. I've built progressively more complex environments — from single-server setups to multi-tier AWS architectures with CI/CD pipelines, container orchestration, and infrastructure as code.

I'm actively pursuing roles in Cloud Engineering, DevOps, Infrastructure Engineering, and Systems Administration.
