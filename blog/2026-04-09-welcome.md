# Starting My Cloud Engineering Journey — In Public

I'm building this site as a living portfolio of my cloud and DevOps work. Instead of keeping notes in private docs, I'm publishing them here — for accountability, for feedback, and to show how I think through technical problems.

<!-- truncate -->

## Why Document Publicly?

- **It forces clarity.** If I can't explain it clearly, I don't understand it well enough.
- **It builds a track record.** Hiring managers can see how I approach problems, not just that I solved them.
- **It helps others.** The troubleshooting guide I write today might save someone hours tomorrow.

## What to Expect

I'll be writing about:

- AWS architecture decisions and trade-offs
- Kubernetes operations and debugging
- Infrastructure as code patterns
- Homelab experiments and failures
- Certification study notes and exam reflections

## The Stack

This site is built with [Docusaurus](https://docusaurus.io/) and deployed to GitHub Pages. The content is Markdown, the config is TypeScript, and the deployment is automated with GitHub Actions. Simple, fast, and version-controlled — exactly how infrastructure should be.
